# Spec: Creator Incentivization — Tier 1 (Tag-on-Share)

**Created:** May 14, 2026
**Status:** Draft — needs engineering review
**Owner:** TBD (split: ZoogIOS + ZoogCloudFunctions)
**Parent plan:** [`growth/q2-q3-26-organic-social.md`](../q2-q3-26-organic-social.md) (forthcoming)
**Related:** Roadmap line item *"incentivize creators to share on Instagram / TikTok"*

---

## Summary

Reward grandparent-creators with a small in-product perk (one premium story unlock, capped at 3/week) when they share a Zoog to Instagram or TikTok with the Zoog hashtag and @ mention pre-filled. Detection is direct — the existing custom share sheet already routes Instagram and TikTok taps through Zoog code, so we know exactly when this happens.

This is **Tier 1 of a three-tier UGC engine**. Tiers 2 (verified public posts) and 3 (direct video submissions) are explicitly out of scope for this spec and will follow if Tier 1 validates.

## Goals

1. **Increase the rate at which Zoogs are shared to public social** (vs. private channels like iMessage / WhatsApp), without reducing total share volume.
2. **Increase the rate at which shared Zoogs include the @zooghq mention and #ZoogMoment hashtag**, so the social-team can find the public posts and amplify them.
3. **Validate the loop economics** — confirm that a free premium story unlock costs less in marginal revenue than the install / awareness value generated by the public share.

## Non-goals (out of scope for Tier 1)

- Verifying that the user actually completed the post on Instagram / TikTok (we reward the intent, not the outcome — Tier 2 will close that loop)
- Capturing the kid's reaction video (Tier 3)
- Native in-product reaction recording (separate Q3/Q4 product spec)
- Auto-detection of public posts via Instagram / TikTok APIs (deferred to Tier 2)
- Cross-platform sharing to Facebook / YouTube / LinkedIn (Tier 1.5 if Instagram + TikTok validate)

## Background

The roadmap had this as a line item with no spec and no owner. The social plan (`growth/q2-q3-26-organic-social.md`) needs a UGC sourcing engine; rather than building a separate marketing-side initiative, this spec carries the existing roadmap item forward and shapes it to serve both goals.

The existing `CustomShareSheet` (`Zoog/App/Features/CustomShareSheet/`) defines `SocialOption.Option` cases for `.instagram` and `.tiktok` and routes each through Zoog code. The detection mechanic for this spec hooks directly into the existing tap handling.

The product data justifies the intervention. From the dataroom (§02): 138K active creators sharing 1.04M videos a year, mostly to private channels. The §03 funnel shows shared content drives the install loop directly. A small bias toward public sharing should compound — public shares reach networks beyond the immediate family, which is exactly the audience expansion the GTM doc has been chasing.

## User flow

1. User finishes recording a Zoog and reaches the share screen
2. User taps **Instagram** or **TikTok** in the social grid
3. **NEW:** Before the deep link to the app fires, Zoog:
   a. Copies a pre-formatted caption to the iOS pasteboard (so user can paste in IG / TikTok)
   b. Shows a brief one-time / dismissible confirmation: *"Tag us with @zooghq and #ZoogMoment to unlock a free premium story this week"* (first 3 times only, then silent)
   c. Logs the share intent event (see Analytics)
   d. Routes user to Instagram / TikTok via the existing `appSchema` deep link
4. **NEW:** On return to Zoog (app foreground after the Instagram / TikTok tap), check eligibility and grant a premium story unlock if the user is under their weekly cap
5. **NEW:** User sees a one-tap confirmation: *"You've unlocked [Story Name]! 🎉"* with a "Browse Premium" CTA

## Pre-filled caption (default)

```
Made this Zoog for [child's name] 💕 Send your own personalized story at zoog.app/start
@zooghq #ZoogMoment
```

Variants:
- If creator has not entered child name: replace with `for my grandkid` / `for my kid` based on `persona` field
- If creator opted out of analytics: omit the personalized intro, keep the brand mention

User can edit the pasted caption freely. We don't enforce inclusion of @zooghq / #ZoogMoment — pasting and editing is the entire interaction.

## Reward economics

- **Reward:** unlock 1 normally-paywalled story (full read access for the unlocking user, no time limit)
- **Cap:** 3 unlocks per user per rolling 7-day window
- **Cost:** marginal revenue impact is approximately the difference between a premium-story-driven subscription conversion and the user reading one premium story for free. With current paywall conversion in the low single digits per story view, the per-grant cost is ~$0.05–$0.20 in expected lost subscription value
- **Upper bound:** assuming 1,000 users hit the cap monthly = 3,000 unlocks × ~$0.20 max cost = ~$600/month in expected lost value, against an upside of even a few hundred new installs from public exposure

## Technical implementation

### iOS (`ZoogIOS`)

**Files affected:**
- `Zoog/App/Features/CustomShareSheet/Views/SocialNetworksGrid/SocialNetworksGridViewModel.swift`
- `Zoog/App/Features/CustomShareSheet/Views/SocialNetworksGrid/Views/SocialNetworkOptionView.swift`
- New: `Zoog/App/Features/CreatorRewards/` module (model, view-model, view for the unlock confirmation)

**Hook point:** `SocialNetworksGridViewModel` already routes the tap on `.instagram` / `.tiktok` to the deep-link handler. Extend that routing to:

1. Build the caption string from a `CaptionBuilder` helper (uses persona + child-name from existing user data)
2. Write to `UIPasteboard.general.string`
3. Log Amplitude event `Tag On Share Triggered` with properties: `platform: "instagram" | "tiktok"`, `recordingId`, `bookId` (if applicable), `userId`
4. Show the dismissible confirmation copy (only on first 3 occurrences per user — track in `UserDefaults`)
5. Continue with existing deep link

**Reward grant on app return:**
- `AppDelegate` `applicationDidBecomeActive` already fires; add a check: was a `Tag On Share Triggered` event fired in the last 60 seconds? If yes, call `GrantTagOnShareReward()` on the `CreatorRewardsService`
- `CreatorRewardsService` calls a Cloud Function (`grantTagOnShareReward`) with `recordingId` + `platform`
- On success response, show the unlock confirmation modal

### Cloud Functions (`ZoogCloudFunctions`)

**New file:** `functions/controllers/grantTagOnShareReward.js`

**Logic:**
1. Authenticate the request (existing auth middleware)
2. Look up the user's recent rewards in Firestore (new collection `creator_rewards`):
   - Query: `creator_rewards` where `userId == X` AND `grantedAt > now - 7d`
   - Count results
3. If count >= 3: return `{ status: "capped", nextAvailableAt: <date> }`
4. Otherwise:
   - Pick a premium story to unlock — algorithm: favor titles the user has not yet purchased, weighted by current featured-tab ranking
   - Write to `creator_rewards`: `{ userId, recordingId, platform, grantedAt, unlockedBookId, expiresAt: null }`
   - Add `unlockedBookId` to the user's `unlockedBooks` array (existing field)
   - Return `{ status: "granted", unlockedBookId, unlockedBookTitle }`

**New collection:** `creator_rewards`
- Fields: `userId`, `recordingId`, `platform` ("instagram" | "tiktok"), `grantedAt` (timestamp), `unlockedBookId`, `weekKey` (yyyy-ww for cap querying)

**New Firestore index:** `creator_rewards (userId asc, grantedAt desc)`

### Analytics (Amplitude)

**New events:**
- `Tag On Share Triggered` — fired in iOS when user taps Instagram or TikTok in the share sheet
  - Properties: `platform`, `recordingId`, `bookId`, `persona`
- `Tag On Share Reward Granted` — fired by Cloud Function on successful grant
  - Properties: `platform`, `recordingId`, `unlockedBookId`, `weekKeyCount` (1, 2, or 3)
- `Tag On Share Reward Capped` — fired by Cloud Function when user is at weekly cap
  - Properties: `platform`, `recordingId`, `nextAvailableAt`
- `Tag On Share Reward Confirmed` — fired in iOS when user dismisses the confirmation modal (lets us measure UX engagement with the reward itself)

### Analytics dashboard

A new chart in the social-team Amplitude dashboard:
- Daily count of `Tag On Share Triggered` by platform
- Daily count of `Tag On Share Reward Granted`
- Conversion ratio: `Granted / Triggered` (should be high — most users will be under cap)
- Funnel: `Tag On Share Triggered → Tag On Share Reward Confirmed → next Start Recording event` — measures whether rewards drive return engagement

## Privacy / safety

- The pre-filled caption uses the child's first name only (no last name, no age, no location). Same data we already use in product copy.
- If the user has opted out of personalized communications (existing flag), the personalized intro is dropped.
- No video data leaves the device through this flow — we only set the iOS pasteboard text, then deep-link to the third-party app.
- No tracking of what the user actually does in Instagram or TikTok.

## Abuse prevention

- **Weekly cap (3)** prevents farming
- **Account age minimum: 7 days** before eligibility (prevents throwaway accounts)
- **Active subscription? eligible.** Subscribers benefit from premium unlocks too — the cap means they're not getting unlimited
- **Recording must be the user's own** (verified via `recordingId` + `userId` match in Firestore) — can't earn rewards by tapping share on someone else's Zoog

Measured 30 days after launch.

|---|---:|---:|
| Share-to-Instagram-or-TikTok rate (% of total shares) | TBD (need pre-launch baseline) | +25% relative |

| Tag On Share Reward Granted / Triggered | n/a | >85% (cap should rarely bite at this volume) |
| `@zooghq` mentions / week (manual count from social team) | TBD | +5–15 visible mentions/week within 60 days |

If `Tag On Share Triggered` is sub-100/week after 30 days, the loop is too weak — investigate placement and copy. If the cap is biting often (low Granted/Triggered), raise the cap.

## Open questions

- [ ] **Should the reward also be available for Facebook / WhatsApp shares?** WhatsApp drives the most private shares but doesn't help public visibility. Probably scope this to Instagram + TikTok only for Tier 1.
- [ ] **What's the right pre-filled caption?** Three variants worth A/B testing post-launch. The CTA URL (`zoog.app/start`) should land on a Custom Product Page (per ASO plan) once CPPs ship.
- [ ] **Does the reward need a redemption flow, or auto-unlock?** Spec assumes auto-unlock — user just sees a confirmation and the unlocked story is now free in their library. Confirm this is acceptable to product/finance.
- [ ] **Do we need to check `subscriptionActive`?** Subscribers don't pay per-story so the unlock is moot for them. Either skip the reward (cleaner) or grant a "thank you" badge / message instead (better UX). Open for product call.
- [ ] **Should the confirmation modal be A/B tested vs. a more prominent celebration?** Current spec is dismissible-low-key; could be a fuller celebration with confetti / share-again CTA.

## What ships next if Tier 1 validates

If Tier 1 hits its 30-day targets, schedule:

- **Tier 2 spec:** verified public post detection (manual submission first, automated mention-API later); higher reward tier (Zoog+ month per verified post)
- **Tier 1.5:** Facebook + Reels integration (lower priority but easy add given the existing `SocialOption` enum)
- **UGC submission landing page** (`getzoog.com/share-your-moment`) for Tier 3 direct video submissions

If Tier 1 underperforms, do not ship Tiers 2 / 3 in current form — the loop economics are wrong and incentives need rethinking.
